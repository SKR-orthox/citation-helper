# v0.8.5 Test Cases (Fixtures + Manual Spot Checks)

This document lists the baseline test set for v0.8.5.

It is primarily a QA checklist based on the snapshot fixture set:
- `tests/fixtures/**.json` (input CitationData)
- `tests/snapshots/**.json` (expected formatted outputs)

Running `npm test` must pass cleanly for this test set.

## Tag conventions (examples)

- DOI_Y / DOI_N
- PAGES_Y / PAGES_N / ELOCATOR
- ISSUE_Y / ISSUE_N
- AUTHORS_1 / AUTHORS_2 / AUTHORS_6 / AUTHORS_12 / AUTHORS_14 / AUTHORS_20 / AUTHORS_21P
- CONSORTIUM_Y / CONSORTIUM_N / CONSORTIUM_TBD
- TYPE_ARTICLE / TYPE_CHAPTER / TYPE_CORRECTION

Notes:
- Some tags are marked TBD when the source metadata is ambiguous or needs confirmation.
- “ELOCATOR” is used for article numbers (e.g., 43663) or e-locators (e.g., e104918).

## What to verify (common for each case)

Snapshot test (automated)
- `npm test` passes (all formatter snapshots match).

Manual spot check (optional, recommended for releases)
- Status in the popup is “Success” on the page.
- Vancouver output looks sane (no broken punctuation when fields are missing).
- APA 7 output looks sane (author separator, year/title/journal/DOI formatting).
- Missing DOI/pages/issue does not break the sentence structure.
- For export formats (BibTeX / CSL-JSON / RIS), the output is generated and copy works.
